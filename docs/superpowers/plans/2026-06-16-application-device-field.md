# Application `device` Field Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional `device` enum (`mobile|web|desktop|pos`, default `web`) to the Application entity, persisted and exposed by the backend (including the app-allowlist snapshot) and editable + filterable in the frontend admin UI.

**Architecture:** `device` is a `VarChar` column on `tb_application` (default `'web'`). The micro-cluster `application.service.ts` persists/reads/normalizes it and includes it in the allowlist snapshot; the gateway swagger DTOs document it; the gateway allowlist store exposes it via a getter. The frontend adds a shared `DeviceType`, threads it through the service, edits it with a `<Select>` on the Edit page, and filters/displays it on the Management page via the existing `advance` where-clause.

**Tech Stack:** Backend — NestJS + Prisma (Postgres), Jest (`bun run test`, `createMockPrisma`). Frontend — React 18 + TypeScript (Vite), shadcn/ui; no unit-test runner (Vitest not set up), so frontend tasks verify via `bun run build` (tsc typecheck + vite build) plus a manual check.

```ts
  private snapshot: Map<string, { allow_all: boolean; apis: Set<string>; device: string }> = new Map();
```

```ts
  replace(items: AppAllowlistSnapshotItem[]): void {
    const next = new Map<string, { allow_all: boolean; apis: Set<string>; device: string }>();
    for (const item of items) {
      next.set(item.id, { allow_all: item.allow_all, apis: new Set(item.apis), device: item.device ?? 'web' });

  }
```

Add this getter to the `AppAllowlistStore` class (e.g. after `isAllowed`):

```ts
   * @returns The device string or undefined / สตริง device หรือ undefined

  getDevice(appId: string): string | undefined {
```
